1 #Config Configuration

Stylesheet configuration

Carapace is built on the Scarab Sass utility framework. This document outlines Carapace's default Scarab configuration. For more information on configuration for the Scarab framework, refer to Scarab's documentation.

To replace the default configuration or define new custom variables, use the define() mixin.

Source: config/_.scss, line 1

1.1 #Config.Baseline Baseline

Configure the global baseline value

@include define( baseline, rem(16) );

The stylesheet's BASELINE is responsible for the sizing of most element and components. You can declare multiples of your baseline in properties throughout your stylesheet using Scarab's bl() mixin.

It is recommended to use rem units for the baseline so that the UI scales with the user's browser font-size setting.

Source: config/baseline.scss, line 1

1.2 #Config.Breakpoints Breakpoints

Configure global breakpoints

@include define( breakpoints, (
    tiny:   em(480),
    small:  em(640),
    medium: em(960),
    large:  em(1440),
    huge:   em(1920)
) );

The BREAKPOINTS configuration map is used in conjunction with Scarab's breakpoint(), type-scale(), query() and responsive() mixins.

Adjust these breakpoints as required by your design. You may define a different number of breakpoints than what is provided by default and use them throughout your project.

Source: config/breakpoints.scss, line 1

1.3 #Config.Durations Durations

Configure global durations

@include define( durations, (
    short:  0.15s,
    medium: 0.4s,
    long:   1.2s
) );

@include define( durations, default, duration(short) );

Configure DURATIONS for consistent transition and animation duration values throughout your project. Use Scarab's duration() getter function to return the value of a named duration.

Carapace sets the default duration to short.

1.4 #Config.Palettes Palettes

Configure global color palettes

@include define( palettes, (
    white: (
        base: #ffffff
    ),
    blue: (
        light: #e3f2fd,
        base: #42a5f5,
        dark: #0d47a1
    ),
    // ... See source file for full configuration
) );

Carapace provides a bunch of default colors, but you should configure these on a per-project basis.

PALETTES should be defined as a map with base color names as keys, and the value of each key should be a map shades to color values.

Use Scarab's palette() getter function to return the color of a color palette.

1.5 #Config.Type-Scale Type Scale

Configure global type scale

@include define( type-scale, (
    title: (
        base:  ( font-size: rem(26), line-height: bl(2) ),
        tiny:  ( font-size: rem(28), line-height: bl(2.5) ),
        large: ( font-size: rem(32), line-height: bl(3) )
    ),
    heading: (
        base:  ( font-size: rem(22), line-height: bl(2) ),
        tiny:  ( font-size: rem(24), line-height: bl(2) ),
        large: ( font-size: rem(28), line-height: bl(2.5) )
    ),
    ... See source file for full configuration
) );

The TYPE-SCALE configuration map is Used in conjunction with Scarab's type-scale() mixin to generate responsive font-size and line-height declarations while maintaining vertical rythmn.

Carapace generates a .fs-#{$size} class for each size name in the TYPE-SCALE configuration map.

1.8 #Config.Breakpoint-Modifiers Breakpoint modifiers

Configure breakpoint-based class modifiers

@include define( breakpoint-mods, (
    tiny,
    small,
    medium,
    large,
    huge
) );

Define a list of BREAKPOINT-MODS that Carapace will use to generate breakpoint "hooks" for each layout and helper class.

An example of a breakpoint "hook" is the .align-right@medium class, which will only apply the .align-right class to an element if the viewport is >= the width of the medium breakpoint.

Source: config/breakpoint-mods.scss, line 1

1.9 #Config.Size-Modifiers Size modifiers

Configure size-based class modifiers

@include define( size-mods, (
    xs,
    s,
    l,
    xl,
    '0'
) );

Define a list of SIZE-MODS that will be used in conjunction with the x and y configuration variables and helper functions. Carapace generates different sizes for some layout and component classes (like .block--xl, .card--s, etc...) for each value in SIZE-MODS.
